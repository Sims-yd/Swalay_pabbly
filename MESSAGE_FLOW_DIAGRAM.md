### âœ… DO

- **Store in database first**, then emit events
- **Target specific users** using userId â†’ socketId mapping
- **Handle reconnection** by re-registering user
- **Use REST APIs** only for initial load and pagination

### `new_message`
**Emitted when**: New message sent or received
**Payload**:
```typescript
{
  id: string,
  chatId: string,
  senderId: string,
  receiverId: string,
  text: string,
  status: "sent" | "delivered" | "read",

  updatedAt: string,
  whatsappMessageId?: string
```

### `message_status_update`
**Emitted when**: Message status changes (delivered/read)
**Payload**:
```typescript
{
  messageId: string,
  whatsappMessageId: string,
  status: "delivered" | "read",
  timestamp: string
}
```

**Payload**:
```typescript

}
```
